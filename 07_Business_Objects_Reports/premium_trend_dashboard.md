# 📈 Premium Trend Dashboard

> A visual summary of premium collection patterns for senior management, underwriting, and finance teams.

---

## 1️⃣ Dashboard Purpose

The **Premium Trend Dashboard** provides a high-level, visual summary of premium collection patterns across **months, quarters, years, regions, agents, and policy types**.

It is designed for **senior management, underwriting teams, and finance departments** to quickly understand revenue trends and make strategic decisions.

> This dashboard is one of the most **visually rich** Business Objects assets in the Data Warehousing Insurance Group project.

---

## 2️⃣ Data Sources Used

| Source | Provides |
|---|---|
| `Policy_Fact` | Premium amounts, coverage, policy duration |
| `Policy_Dim` | Policy type, status, attributes |
| `Agent_Dim` | Agent region and performance |
| `Customer_Dim` | Customer demographics |
| `Date_Dim` | Calendar hierarchy for premium trend visualization |

---

## 3️⃣ Key Dashboard Components

### 🅰️ Monthly Premium Trend Chart
- Premium collected each month
- Seasonal variations
- Month-over-month growth

### 🅱️ Quarterly Premium Summary
- Total premium per quarter
- Quarter-over-quarter comparison
- Trend indicators (📈 up / 📉 down)

### 🅲️ Year-Over-Year Premium Comparison
- Premium totals for each year
- Growth percentage
- Historical premium trend line

### 🅳️ Region-Wise Premium Contribution
- Premium generated by each region
- Regional ranking
- 🗺️ Heat map visualization

### 🅴️ Policy Type Premium Distribution
- Premium contribution by policy type
- Life vs Auto vs Health vs Property
- Pie chart or bar chart representation

### 🅵️ Agent Premium Contribution
- Premium generated per agent
- 🏆 Top 10 agents
- ⚠️ Underperforming agents

---

## 5️⃣ Sample BO Query Logic (Semantic Layer)

**Policy → Fact Join**
```sql
Policy_Dim.policy_key = Policy_Fact.policy_key
```

**Date Dimension Join**
```sql
Policy_Fact.date_key = Date_Dim.date_key
```

**Agent Join**
```sql
Policy_Fact.agent_key = Agent_Dim.agent_key
```
